# Техническое задание: Система скинов оружия (Часть 2 - Скины)

## Обзор

Создать систему скинов для оружия в игре "Low Poly Shooter Pack v4.3". Скины позволяют изменять внешний вид оружия без изменения его характеристик.

## Требования

### 1. Система редкости скинов

**Уровни редкости:**
- **Common** (Обычный) - базовый цвет, низкая стоимость
- **Rare** (Редкий) - улучшенный цвет, средняя стоимость
- **Epic** (Эпический) - уникальный цвет, высокая стоимость
- **Legendary** (Легендарный) - эксклюзивный дизайн, очень высокая стоимость

**Цветовая кодировка редкости:**
- Common: Серый/Белый
- Rare: Синий
- Epic: Фиолетовый
- Legendary: Золотой/Оранжевый

### 2. Конфигурация скинов

**Параметры скина:**
- Уникальный идентификатор скина (skin_id)
- Название скина
- Редкость скина
- Материал для применения к оружию
- Стоимость скина
- Описание скина
- Превью изображение (опционально)

**Связь с оружием:**
- Каждый скин привязан к конкретному оружию
- Одно оружие может иметь несколько скинов
- Скин не может быть применён к другому оружию

### 3. Магазин скинов

**Функционал:**
- Отображение списка доступных скинов для выбранного оружия
- Просмотр превью скина на оружии
- Покупка скина за игровую валюту
- Применение скина к оружию
- Отображение статуса (куплено/не куплено/выбрано)

**UI элементы:**
- Список скинов (Grid Layout)
- Карточка скина с превью
- Индикатор редкости скина
- Цена скина
- Кнопка "Купить" (если не куплено)
- Кнопка "Выбрать" (если куплено)
- Индикатор выбранного скина

**Логика:**
- Проверка достаточности валюты перед покупкой
- Списание валюты при покупке
- Сохранение купленных скинов
- Сохранение выбранного скина для оружия
- Автоматическое применение выбранного скина в игре

### 4. Система материалов скинов

**Требования к материалам:**
- Материалы должны быть совместимы с шейдерами оружия
- Поддержка изменения цвета текстуры
- Поддержка изменения нормалей (опционально)
- Поддержка изменения металличности (опционально)

**Конфигурация материалов:**
- Основной цвет (Albedo)
- Цвет металличности (Metallic)
- Гладкость (Smoothness)
- Нормальная карта (Normal Map)

### 5. Сохранение прогресса

**Формат данных (JSON):**
```json
{
  "weapons": {
    "weapon_id_1": {
      "purchased": true,
      "selected": true,
      "skins": {
        "skin_id_1": {
          "purchased": true,
          "selected": true
        },
        "skin_id_2": {
          "purchased": false,
          "selected": false
        }
      }
    }
  }
}
```

### 6. Интеграция с существующими системами

**Интеграция с Weapon:**
- Применение материала скина к оружию
- Изменение внешнего вида оружия в реальном времени

**Интеграция с Inventory:**
- Отображение выбранного скина в инвентаре
- Применение скина при выборе оружия

## Технические требования

### Язык программирования
- C#

### Unity версии
- Совместимость с Unity 2020.3+

### Архитектура

**Слои:**
1. **Data Layer** - ScriptableObject конфигурации скинов
2. **Services Layer** - Сервис магазина скинов
3. **UI Layer** - Компоненты интерфейса
4. **Persistence Layer** - Сохранение/загрузка данных

### Файлы для создания

**Data Layer:**
- `SkinData.cs` - ScriptableObject для конфигурации скина
- `SkinMaterialData.cs` - ScriptableObject для материалов скина

**Services Layer:**
- `ISkinShopService.cs` - Интерфейс сервиса магазина скинов
- `SkinShopService.cs` - Реализация сервиса магазина скинов

**UI Layer:**
- `SkinsShopUI.cs` - Основной UI магазина скинов
- `SkinItemUI.cs` - UI элемент скина в списке
- `SkinPreview3D.cs` - 3D предпросмотр скина на оружии

**Persistence Layer:**
- `SkinShopSaveSystem.cs` - Система сохранения для магазина скинов

## Приоритет задач

1. **Высокий приоритет:**
   - Создание ScriptableObject для конфигурации скинов
